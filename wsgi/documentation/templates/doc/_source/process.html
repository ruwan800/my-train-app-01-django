Documentation of Process¶

executable¶

process.executable.startProcess(text)[source]¶

Starts thread to execute series of functions mentioned in each Stage belongs to the Process identified by given text. State of the executing Stage and progress information are recorded in Progress.

models¶

class process.models.Process(*args, **kwargs)[source]¶

A Process responsible for executing series of functions kept on each Stage. Each Stage‘s function is executed using Stage.execute(). Information related to execution is preserved in Progress. Breakpoints can be setup by modifying Progress. Each Process have it’s own Progress instance.

class process.models.Progress(*args, **kwargs)[source]¶

Progress keeps information about currently executing function, progress information and break-point information.

  • process

  • stage

  • progress -current progress of the stage

  • complete

    Sets number of steps in the stage. this will set automatically if the stage method implemented with setProgressBoundary().

  • break_stage

    If the stage is not empty the process will execute that stage as the last stage before terminate.

  • break_point

    This can be set manually along with break_stage. The Stage will terminated after executing step mentioned in the break_point.

process.models.incrementProgress(text)[source]¶

This function increments the value of completed steps in the current Stage of Process identified by text. This function should be implemented in steps that takes considerably long time to complete. That will help to track down the progress of the current Stage. This function will raise an exception if incrementing value passed the breakpoint defined within Progress. This method should not use twice within single step to avoid incrementing step count twice. However it will not affects execution of the function. But it will results erroneous step count and affects the breakpoint

setProgressBoundary() and getCurrentProgress() should also be implemented along with this function.
